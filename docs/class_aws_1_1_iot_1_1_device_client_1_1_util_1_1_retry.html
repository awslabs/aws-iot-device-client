Performs an exponential backoff of the provided function based on the specified ExponentialRetryConfig.

In the event of throttling by IoT Core APIs, such as when we perform UpdateJobExecution within the Jobs feature, it is necessary to perform an exponential backoff to improve the chances of receiving a success response.

Parameters
configthe ExponentialRetryConfig specifying whether the function should be retried
retryableFunctionthe function to retry. This function should return a bool indicating whether it is successful or not, since this indicator is what will determine whether the function is retried or not.
onCompletea callback function which will be executed once this function is finished attempting retries
Returns
a bool representing whether the retryableFunction was successful or not
